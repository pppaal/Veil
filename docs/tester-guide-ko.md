# VEIL 베타 테스터 가이드

이 문서는 VEIL Private Beta 에 초대받은 친구/지인을 위한 안내입니다.
"버그 신고는 어디로?", "안전한가?", "내 메시지 어디 가나?" 같은 질문에
한 번에 답하려고 만들었어요.

## 1. 사전 준비

- 모바일 브라우저 (Chrome, Safari) 또는 데스크톱 브라우저
- 운영자가 알려준 데모 URL (예: `https://veil-beta.example.com/demo`)
- (선택) PWA 로 홈 화면에 추가 → 알림창처럼 띄우려면

## 2. 첫 사용

1. 데모 URL 접속
2. "PRIVATE BETA" 빨간 배너 확인 → 닫지 말고 한번 읽어주세요
3. **표시 이름** + **핸들** 입력해서 "새 핸들로 시작"
   - 핸들은 한 번 정하면 못 바꿉니다
   - 소문자/숫자/_ 만 가능
4. 친구한테 본인 핸들 (예: `@alice`) 알려주기
5. "새 대화" 누르고 친구 핸들 입력 → 대화 시작

## 3. 꼭 알아둘 것 (중요)

### 백업 없음, 복구 없음

- 비밀번호 재설정 같은 거 **없습니다**
- 브라우저 캐시 지우면 → 메시지 + 핸들 둘 다 영구 손실
- 다른 기기로 옮기려면 **이 기기에서 직접** 이전 절차 (베타에선 미구현)

이건 버그가 아니라 **설계**입니다. 서버에 키가 없어서 우리가 복구해줄
수가 없어요. 카톡/텔레그램이랑 다른 점.

### 서버는 ciphertext 만 봅니다

- 메시지 본문은 운영자가 못 봅니다
- DB 직접 열어도 `ciphertext` (의미 없는 base64) 만 보임
- 첨부 파일도 client-side 암호화 (베타에선 첨부 비활성)

### 외부 크립토 감사 미완료

- 종단간 암호화는 동작하지만 외부 보안 감사가 아직 안 끝났어요
- 베타 = "기능 동작 확인" 단계. 진지한 비밀은 정식 출시까지 보내지
  말아주세요

## 4. 자주 마주칠 상황

### "🔒 복호화 실패" 가 모든 메시지에 떠요

- 한쪽이 키를 회전한 직후 잠깐 나타나는 정상 동작
- 3회 연속 실패하면 자동으로 peer 키 캐시 갱신 시도
- 그래도 계속이면 → 양쪽 다 "이 브라우저의 키 · 캐시 전부 삭제"
  하고 재가입

### 새 메시지가 안 와요

- 우상단에 "연결됨" 인지 확인
- "연결 끊김" 이면 자동 재연결까지 5초
- 메뉴 → "연결 상태" 로 WebSocket 인지 폴링인지 확인

### 폰을 바꿨어요

베타에선 디바이스 이전이 미구현입니다. 지금은:
- 새 폰에서 **새 핸들** 로 가입
- 이전 핸들로는 다시 못 들어갑니다 (보안 정책)

정식 출시 시 transfer 기능 추가 예정.

### 친구가 차단되거나 나를 차단한 것 같아요

- 차단된 사용자는 "그 핸들 없습니다" 처럼 보입니다
  (블록 상태를 노출하지 않는 게 보안 원칙)
- 운영자한테 "왜 안 보여요?" 물어도 운영자도 모릅니다 (정상)

## 5. 버그 신고

베타 중 발견한 버그는 운영자한테 직접:
- 어디서 (URL, 화면)
- 무엇을 했는지 (단계 1, 2, 3)
- 어떻게 됐는지 (기대 vs 실제)
- 가능하면 스크린샷

⚠️ **메시지 본문이나 핸들은 보내지 마세요.** 운영자도 그걸 보면 안 됩니다.
"@bob 한테 메시지 보냈는데 빨간 X 떠요" 정도면 충분해요.

## 6. 베타 종료 시

베타 종료일은 운영자가 미리 공지합니다 (보통 4-8주). 종료 시:
- 모든 메시지/계정/키 영구 삭제
- 정식 출시는 별도 가입 (기존 핸들 재사용 보장 없음)

## 7. 신고

- 스팸/괴롭힘: 운영자 (`abuse@veil.example`) 한테 가해자 핸들 + 사건
  일시 보고
- 운영자는 메시지 본문은 못 봅니다. 메타데이터 (누가 누구한테 언제)
  로만 판단해요
- 처리 정책: `docs/ops/abuse-triage.md`

## 8. 자주 묻는 질문

**Q. 안전한가요?**
A. 아키텍처는 Signal/iMessage 와 같은 X25519+AES-256-GCM 입니다.
   다만 외부 크립토 감사 전이라 베타 단계. "기능 확인" 용으로만.

**Q. 텔레그램/카톡 대체할 수 있어요?**
A. 베타 단계에선 안 됩니다 — 첨부, 통화, 푸시 알림이 부분 구현. 정식
   출시 시 1:1 메시지+그룹+첨부+통화+푸시 모두 지원 예정.

**Q. 내 데이터 어디 있어요?**
A. 메시지 본문 키는 **이 브라우저** 에만 있습니다. 서버에는 ciphertext +
   메타데이터 (핸들, 가입 시각, 메시지 시각). EU 거주자는 GDPR 데이터
   주체 권리 청구 가능 (`docs/ops/data-subject-request.md`).

**Q. 운영자가 내 메시지 볼 수 있어요?**
A. 못 봅니다. 베타라서 코드 베이스 본인이 직접 확인 가능 (오픈소스):
   메시지 본문 복호화 코드는 클라이언트에만 있습니다.

**Q. 이 베타 사이트 24/7 살아있나요?**
A. 아니요. 운영자 노트북이 켜져있을 때만 (Cloudflare Tunnel 구조).
   접속 안 되면 운영자한테 핑.

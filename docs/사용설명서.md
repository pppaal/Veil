# VEIL 사용 설명서

> 한 문서로 보는 VEIL — 이게 뭔지, 어떻게 돌리는지, 어떻게 쓰는지,
> 어떻게 배포하는지. 개발자·운영자·테스터 모두를 위한 한국어 가이드.

---

## 1. VEIL 이 뭐예요?

**백업 없음 · 복구 없음 · 누출 없음** 을 핵심 가치로 하는 종단간 암호화
메신저입니다.

- 서버는 **암호문(ciphertext)만** 봅니다. 평문은 절대 갖지 않아요.
- **백업·복구·비밀번호 재설정이 없습니다.** 기기를 잃으면 메시지·핸들
  영구 손실. 이건 버그가 아니라 **설계**예요.
- 신분은 **기기에 묶입니다.** 다른 기기로 옮기려면 직접 이전(transfer)
  절차를 밟아야 해요.

이 원칙들은 문서로만 적힌 게 아니라 `scripts/policy-check.mjs` 가 CI
에서 강제합니다 — 누가 복구 기능을 코드에 추가하면 빌드가 빨강이 됩니다.

### 카카오톡과 뭐가 다른가요?

| | VEIL | 카카오톡 |
|---|---|---|
| 서버에 평문 저장 | ❌ 절대 안 함 | ✅ 함 |
| 백업/복구 | ❌ 없음 | ✅ 톡서랍 |
| 전화번호 | ❌ 핸들만 | ✅ 필수 |
| 압수수색 시 | ciphertext 만 나옴 | 평문 제공 가능 |

카카오톡 **대체용이 아니라**, "카톡으론 못 보낼 얘기를 보내는 두 번째
메신저" 입니다.

## 2. 빠른 시작 (3가지 방법)

### 방법 A — 내 컴퓨터에서 (개발/테스트)

```bash
# 도커 한 방
docker compose -f docker-compose.demo.yml up

# 브라우저에서
open http://localhost:3000/demo/
```

같은 와이파이의 폰에서 보려면 `http://<노트북-LAN-IP>:3000/demo/`.
자세히: [`docs/phone-access.md`](phone-access.md)

### 방법 B — 친구한테 공개 (Cloudflare Tunnel)

노트북 켜져 있는 동안만 동작. 무료 HTTPS.

```bash
export CLOUDFLARE_TUNNEL_TOKEN=<Zero Trust 대시보드 토큰>
pnpm demo:tunnel:up
```

자세히: [`docs/cloudflare-tunnel-deploy.md`](cloudflare-tunnel-deploy.md)

### 방법 C — VPS 24/7 운영 (공개 베타)

```bash
cp .env.prod.example .env.prod   # 편집 후
pnpm demo:prod:up                # Caddy TLS + 전체 스택
pnpm demo:prod:obs               # Prometheus + Grafana (선택)
```

자세히 (90분 절차): [`docs/vps-deploy-runbook.md`](vps-deploy-runbook.md)

## 3. 개발 환경 셋업

```bash
# 1. Node 22.20.0 (.nvmrc 참고)
# 2. 환경변수
cp .env.example .env
cp apps/api/.env.example apps/api/.env

# 3. 의존성
pnpm install

# 4. 인프라 (Postgres + Redis + MinIO)
pnpm docker:up

# 5. Prisma 클라이언트
pnpm db:generate

# 6. API 개발 서버
pnpm dev:api

# 7. 웹 데모: http://localhost:3000/demo/
```

모바일 (Flutter 설치 후):
```bash
pnpm dev:mobile:api       # 안드로이드 에뮬레이터
pnpm dev:mobile:desktop   # 윈도우 데스크탑 점검
```

> ⚠️ `VEIL_JWT_SECRET` 은 최소 32바이트. `openssl rand -base64 32` 로 생성.

## 4. 웹 데모 기능 사용법

### 가입 / 로그인
1. 표시 이름 + 핸들 입력 → "새 핸들로 시작"
2. 핸들은 소문자/숫자/_, 한 번 정하면 못 바꿈
3. 저장된 세션 있으면 "계속" 으로 복원

### 메시지 보내기
- 입력창에 텍스트 → Enter (Shift+Enter 줄바꿈)
- 마크다운: `*굵게*`, `_기울임_`, `` `코드` ``, ` ```코드블록``` `
- URL 자동 링크, `@핸들` 멘션 자동완성

### 메시지 액션 (우클릭 또는 길게 누르기)
| 액션 | 설명 |
|---|---|
| 👍 빠른 반응 | 6개 이모지 |
| ↩ 답장 | 인용 후 전송 |
| 📋 복사 | 본문 복사 |
| ✏️ 수정 | 내 메시지만, 재암호화 |
| 🗑 삭제 | 내 메시지만, 5초 되돌리기 |
| 🔁 전달 | 다른 대화로 |
| 🚫 차단 | 상대 메시지만 |
| 🚨 신고 | 운영자에게 |

### 음성 메시지
- 🎤 버튼 **길게 누르기** → 녹음 (최대 30초) → 떼면 전송
- 너무 짧으면(0.4초 미만) 취소

### 이미지 / 파일
- 📷 이미지: 선택 또는 드래그-드롭 → 미리보기 확인 → 전송
- 📎 파일: PDF/비디오/zip 등 25MB까지, 다운로드 chip 으로 표시
- 모두 클라이언트에서 AES-256-GCM 암호화 후 업로드

### 검색 / 멘션
- `Ctrl/Cmd+F` 또는 🔍 → 대화 내부 검색 (Enter 다음, Shift+Enter 이전)
- `@` 입력 → 대화 참여자 자동완성 (↑↓ Enter)

### 카카오톡 가져오기
1. 메뉴 → "📥 카카오톡 가져오기"
2. 카톡 → 채팅방 → 설정 → 대화 내용 내보내기 → .txt
3. .txt 떨어뜨리기 → 미리보기 → "아카이브로 저장"
4. 사이드바 "카카오 아카이브" 섹션에 **읽기 전용**으로 보임
5. ⚠️ 바이트가 절대 서버로 안 감 (100% 브라우저 안)

### 설정 (메뉴 → ⚙️ 설정)
- 테마: 다크 / 라이트
- 언어: 한국어 / English / 日本語 (재로드)
- 사운드: 수신/전송 톤 (기본 꺼짐)
- 알림: 브라우저 권한
- 차단된 사용자 관리

### 키보드 단축키
| 키 | 동작 |
|---|---|
| `Ctrl/Cmd+K` | 대화 검색 |
| `Ctrl/Cmd+N` | 새 대화 |
| `Ctrl/Cmd+F` | 대화 안에서 찾기 |
| `Ctrl/Cmd+/` | 도움말 |
| `Ctrl/Cmd+Shift+S` | 사운드 토글 |
| `Esc` | 다이얼로그/메뉴 닫기 |

### 디바이스 이전
- **구 기기**: 메뉴 → "다른 기기로 이전" → 코드 표시 → 새 기기가
  클레임하면 지문 확인 후 승인 (구 기기 자동 로그아웃)
- **신 기기**: "기존 기기에서 이전" → 코드 붙여넣기 → 지문 확인 → 구
  기기 승인 대기 → 자동 완료

## 5. 아키텍처 요약

```
┌─────────────┐   ┌─────────────┐   ┌─────────────┐
│ 웹 데모      │   │ Flutter 앱  │   │ (향후 클라) │
│ (바닐라 JS) │   │ (모바일)    │   │             │
└──────┬──────┘   └──────┬──────┘   └──────┬──────┘
       │ ciphertext only  │                 │
       └──────────────────┼─────────────────┘
                          ▼
                ┌───────────────────┐
                │  NestJS API (/v1) │  19 모듈
                │  + WebSocket      │  JWT 1h + JTI blacklist
                └─────────┬─────────┘
            ┌─────────────┼─────────────┐
            ▼             ▼             ▼
        Postgres       Redis         MinIO
       (ciphertext)  (ephemeral)  (encrypted blobs)
```

- **크립토**: 모바일은 X25519 ECDH + HKDF-SHA256 + AES-256-GCM +
  Double Ratchet (forward secrecy + post-compromise security). 웹은
  단순 ECDH per-message (envelope v3 통합 예정).
- **인증**: Ed25519 challenge/verify + JTI blacklist + 로그아웃 시 WS
  강제 끊기.
- **배포**: docker-compose 3종 (demo / alpha / prod) + Caddy TLS +
  Prometheus/Grafana.

자세히: [`docs/architecture.md`](architecture.md),
[`docs/threat-model.md`](threat-model.md)

- 웹 Vitest **40** (markdown 20 + format 12 + kakao-import 8)

## 9. 자주 묻는 질문

**Q. 정식 출시해도 되나요?**
A. 아직요. `VEIL_AUDITED_CRYPTO_ATTESTED=false` 라 production 부팅이
거부됩니다. 외부 감사 후에만. 친구/공개 베타는 가능.

**Q. 메시지를 정말 서버가 못 보나요?**
A. 네. DB 직접 열어도 ciphertext (의미 없는 base64) 만 나옵니다.
   확인: `docs/cloudflare-tunnel-deploy.md` 의 DB 직접 조회 섹션.

**Q. 폰 바꾸면 어떻게 되나요?**
A. 베타에선 디바이스 이전(transfer) 으로 옮길 수 있어요. 단 구 기기가
   살아있어야 합니다. 구 기기 잃으면 영구 손실.

**Q. 크립토는 Signal 코드인가요?**
A. 아니요. Signal 의 공개 알고리즘(Double Ratchet)을 직접 구현했어요.
   libsignal 라이브러리는 안 씁니다. 그래서 외부 감사가 더 중요.

**Q. 카톡 대체할 수 있나요?**
A. 못 합니다 (네트워크 효과 + 생태계). "두 번째 메신저" 포지션.

# VEIL — Play 스토어 스크린샷 캡션 (한국어)

스토어 전환율의 대부분은 **첫 2~3장**이 결정합니다. 사용자는 설명을 읽지
않고 스크린샷만 넘겨봅니다. 각 컷은 **큰 헤드라인 1줄 + 보조설명 1줄**로,
3초 안에 차별점이 읽혀야 합니다.

## 제작 가이드 (Google Play)
- 휴대폰 스크린샷 권장: **1080 × 1920 이상**, 세로, PNG/JPEG (최소 2장, 최대 8장).
- 헤드라인은 화면 상단 1/3에 크게. 앱 UI는 하단에 살짝 잘리게 배치(스크롤 암시).
- 톤: 다크 배경 + 단정한 산세리프. 브랜드 컬러 1색 포인트.
- **첫 컷에 핵심 가치**, 마지막 컷에 철학/행동유도(CTA).
- 6컷 전부 같은 템플릿(헤드라인 위치·폰트·여백)으로 통일 → 시리즈처럼 보이게.

## 6컷 구성

### 1컷 — 핵심 가치 (가장 중요)
- **헤드라인:** 서버도 못 읽는 메시지
- **보조:** 모든 대화는 기기에서 종단간 암호화됩니다
- **화면:** 채팅 화면(`chat`) — 자물쇠 아이콘이 보이는 대화
- **노트:** 첫인상 = 전부. "서버도 못 읽는다"가 한 줄 후킹.

### 2컷 — 전화번호 없음
- **헤드라인:** 전화번호 없이 시작
- **보조:** 핸들 하나면 끝. 번호도, 실명도 묻지 않습니다
- **화면:** 온보딩/핸들 선택(`onboarding`)
- **노트:** 카톡과 가장 명확히 갈리는 지점 → 2번째로 배치.

### 3컷 — 광고·추적 0
- **헤드라인:** 광고도, 추적도 없음
- **보조:** 제3자 분석 0. 당신의 데이터를 팔지 않습니다
- **화면:** 대화 목록(`conversations`) — 깔끔한 빈 UI 강조
- **노트:** "없음"의 연속이 곧 제품 정체성.

### 4컷 — 진짜 암호화
- **헤드라인:** 군용급 암호화
- **보조:** X25519 · AES-256-GCM · Ed25519
- **화면:** 보안 상태 화면(`security_status` / `security`)
- **노트:** 기술 신뢰. 숫자·약어가 오히려 진짜처럼 보임.

### 5컷 — 앱 잠금
- **헤드라인:** 내 손 안에서만 열림
- **보조:** PIN·생체인증으로 앱 자체를 잠급니다
- **화면:** 앱 잠금(`app_lock`) — 생체인증 프롬프트
- **노트:** 기기를 뺏겨도 안전 = 일상적 안심 포인트.

### 6컷 — 철학 + CTA
- **헤드라인:** 복구 불가. 그게 안전입니다
- **보조:** 기록이 남지 않는 메신저. 지금 시작하세요
- **화면:** 브랜드 로고 + 다운로드 유도(또는 기기 전송 `device_transfer`)
- **노트:** 마지막 컷은 행동유도. "복구 불가"를 약점이 아닌 가치로 재정의.
